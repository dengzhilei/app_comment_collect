# 游戏商店评论分析项目计划

## 项目概述

**目标**：采集并分析特定游戏在iOS和Google Play商店的评论数据，提取玩家情绪、游戏亮点、痛点等信息，为游戏设计提供数据支持。

**目标游戏**：
- Carnival Tycoon
- TopTycoon
- Animals & Coins
- Fish of Fortune

**时间范围**：最近一年（2024年）

**平台**：iOS App Store、Google Play Store

---

## 技术架构

### 1. 数据采集层
- **工具选择**：
  - App Store：使用 `app-store-scraper` 或 `itunes-app-scraper`
  - Google Play：使用 `google-play-scraper` 或 `play-scraper`
  - 备选：手动API调用或爬虫方案

### 2. 数据存储层
- **格式**：JSON/CSV
- **数据库**：SQLite（轻量级）或 PostgreSQL（如需扩展）
- **文件结构**：
  ```
  data/
    ├── raw/           # 原始评论数据
    ├── processed/     # 清洗后的数据
    └── analysis/      # 分析结果
  ```

### 3. 数据分析层
- **语言**：Python
- **工具库**：
  - 文本分析：`jieba`（中文分词）、`nltk`/`spaCy`（英文分析）
  - 情感分析：`TextBlob`、`VADER`、`snownlp`（中文）
  - 数据可视化：`matplotlib`、`seaborn`、`plotly`
  - 数据处理：`pandas`、`numpy`

### 4. 报告生成层
- **输出格式**：Markdown报告 + 可视化图表
- **内容结构**：
  - 数据概览
  - 情感分析结果
  - 关键词提取
  - 玩家反馈分类（优点/缺点/建议）
  - 趋势分析

## 实施步骤

### 阶段一：环境搭建与工具准备（1-2天）

1. **项目初始化**
   - 创建项目目录结构
   - 配置Python虚拟环境
   - 安装依赖包

2. **数据采集工具配置**
   - 测试App Store评论采集
   - 测试Google Play评论采集
   - 验证数据格式和完整性

3. **数据存储设计**
   - 设计数据表结构
   - 创建数据库/文件存储方案

### 阶段二：数据采集（2-3天）

1. **游戏信息确认**
   - 确认每个游戏在App Store和Google Play的准确名称和ID
   - 验证游戏是否存在及可访问性

2. **批量采集评论**
   - 实现采集脚本
   - 设置采集时间范围（最近一年）
   - 处理采集异常和重试机制
   - 数据去重和验证

3. **数据质量检查**
   - 检查数据完整性
   - 验证时间范围正确性
   - 统计采集数据量

### 阶段三：数据清洗与预处理（1-2天）

1. **数据清洗**
   - 去除重复评论
   - 处理缺失值
   - 统一时间格式
   - 标准化评分格式

2. **文本预处理**
   - 去除特殊字符和HTML标签
   - 处理多语言文本（中英文）
   - 文本标准化

3. **数据标注**
   - 按游戏分类
   - 按平台分类（iOS/Android）
   - 按时间分段（可选）

### 阶段四：数据分析（3-4天）

1. **基础统计分析**
   - 评论数量统计
   - 评分分布分析
   - 时间趋势分析
   - 平台对比分析

2. **情感分析**
   - 整体情感倾向（正面/中性/负面）
   - 情感强度分析
   - 情感趋势变化

3. **关键词提取**
   - 高频词汇提取
   - 主题词提取
   - 玩家常用表达分析

4. **内容分类分析**
   - 自动分类：游戏玩法、画面、音效、bug、付费、更新等
   - 优点提取
   - 缺点提取
   - 建议提取

### 阶段五：深度分析与洞察提取（2-3天）

1. **玩家情绪分析**
   - 兴奋点识别（什么让玩家兴奋）
   - 痛点识别（什么让玩家不满）
   - 情绪变化趋势

2. **游戏设计洞察**
   - 玩家喜欢的游戏机制
   - 玩家期望的功能
   - 玩家反馈的改进建议
   - 竞品对比分析

3. **可视化展示**
   - 情感分布饼图/柱状图
   - 关键词云图
   - 时间趋势折线图
   - 分类统计图表

### 阶段六：报告生成与总结（1-2天）

1. **生成分析报告**
   - 综合报告（所有游戏）
   - 单游戏详细报告
   - 对比分析报告

2. **输出设计建议**
   - 基于数据的游戏设计建议
   - 玩家期望功能清单
   - 需要避免的问题清单

## 关键技术点

### 1. 数据采集
- **App Store**：可能需要处理反爬虫机制，考虑使用代理或API
- **Google Play**：相对容易采集，注意请求频率限制
- **数据量预估**：每个游戏每个平台预计1000-5000条评论

### 2. 多语言处理
- 中文评论：使用 `jieba` + `snownlp`
- 英文评论：使用 `nltk` + `TextBlob`
- 混合语言：需要语言检测和分别处理

### 3. 情感分析
- 使用预训练模型或规则库
- 考虑游戏领域的特殊表达（如"肝"、"氪金"等游戏术语）

### 4. 内容分类
- 可以使用关键词匹配 + 规则分类
- 或使用简单的机器学习分类器

## 风险与应对

### 风险1：数据采集受限
- **应对**：准备多个采集工具备选方案，考虑手动补充

### 风险2：评论数据量不足
- **应对**：扩大时间范围或增加采集频率

### 风险3：多语言分析准确性
- **应对**：使用多个分析工具交叉验证，人工抽样检查

### 风险4：游戏名称匹配错误
- **应对**：先手动验证游戏ID，确保采集正确的游戏
