# v1 制作交互设计定稿

## 1. 布局总览（上下结构，竖屏）

- **区域 A（上）**：基础原料区，紧凑。
- **区域 B（下）**：烹饪合成区，核心操作与浏览。

---

## 2. 区域 A：基础原料区

- **Tab**：保留 小麦 / 乳制品 / 蔬菜 / 肉类 四类，与现有「制作」页 Tab 一致。
- **内容**：当前 Tab 下，**横向**展示该链 Lv1 → Lv2 → Lv3（及 Lv4 若有），每级显示：图标 + 名称 + 拥有数量 + 合成按钮（若已解锁且材料够）。
- **约束**：区域高度压缩，不抢下方合成区视线；可考虑 Tab 与链同一行或链可横向滑动。

---

## 3. 区域 B：烹饪合成区

### 3.1 顶部工具栏（Filter Bar）

| 元素 | 说明 |
|------|------|
| 分类标签 | [全部] [早餐] [主菜] [甜点]（按用餐场景，与「按原料」的 Tab 互补） |
| 功能开关 | 勾选「仅显示可制作」时，只显示材料齐且已解锁的菜品 |

- 实现要点：每条交叉配方需配置 `category: 'breakfast'|'main'|'dessert'`（或全部），列表按分类过滤；「可制作」= 厨师等级 ≥ 解锁等级 且 背包材料满足 needs。

### 3.2 核心内容区（4 列 × N 行网格）

- **布局**：4 列网格，纵向滚动；每格为一张「菜品卡片」。
- **单格内容**：
  - 中心：大号菜品图标（如 🍔）。
  - 左上：等级标签（如 Lv.3）。
  - 右上：状态角标 —— 🟢 可制作 / 🔒 未解锁（等级或链未开放）。
  - 底部：菜名（如「芝士汉堡」）。
- 点击格子：打开**详情弹窗**，不直接制作。

### 3.3 详情弹窗（Detail Modal）

- **触发**：点击任意菜品格子。
- **内容**：
  - 顶部：大图 + 菜品名称。
  - 中部：配方公式，每行一项材料，格式：`[图标] 名称 (当前拥有/需要) ✅/❌`，缺失项用红色或 ❌ 强调。
  - 底部按钮：
    - 材料齐：亮色 **[ 制作 ]**，点击扣材料、加成品、关闭弹窗。
    - 材料缺：灰色 **[ 材料不足 ]**，禁用点击，可配文案如「缺芝士」。
- **与等级关系**：未解锁菜品在列表中可显示为 🔒，点开详情可展示配方但按钮为「未解锁」并说明所需厨师等级。

---

## 4. 与现有实现的对应

- 当前「制作」页：按链 Tab + 链内合成 + 底部交叉合成列表。定稿保留「按链的原料区」，将「交叉合成」改为**按场景分类 + 网格卡片 + 详情弹窗内制作**。
- 现有 `CROSS_RECIPES` 结构可扩展：增加 `category`、`unlockChefLevel`；新 UI 从该列表生成网格与弹窗，不再依赖当前 craft 页的列表 DOM 结构。

---

## 5. 实施顺序建议

1. 先做**数据与逻辑**：厨师等级、经验、链/配方解锁（见 v1-节奏与等级定稿.md）。
2. 再做**制作页布局**：上下分 A/B，B 区先改为 4 列网格 + 从 CROSS_RECIPES 渲染卡片。
3. 再做**筛选与详情**：分类筛选、「仅显示可制作」、点击格子打开详情弹窗与制作按钮。
4. 最后**打磨**：状态角标、缺料提示、未解锁提示。

这样节奏设计与制作交互可逐步落地，且与现有合成链配置一致。
