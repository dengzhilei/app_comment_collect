# v7 棋盘获得表现 - 设计方案

## 目标
- 获得信息放在**棋盘上方**醒目位置，形成清晰的「奖励时刻」。
- 概率差异化：**大概率基础、小概率 2 级、很小概率 3 级**，2/3 级有**幸运表现**。

## 1. 布局
- **本次获得**区域：位于棋盘（.board-wrap）**正上方**，在走格子面板内。
- 默认态：显示「—」或「掷骰落格后显示」。
- 掷骰落格后：立即更新为「emoji + 物品名 × 数量」，卡片式、略大字号。

## 2. 概率（config）
- Lv1 约 **78%**，Lv2 约 **18%**，Lv3 约 **4%**（可微调）。
- 使 2/3 级明显稀有，出的时候有「中奖感」。

## 3. 幸运表现（Lv2 / Lv3）
- **Lv1**：普通样式，仅显示获得内容。
- **Lv2**：区域加「幸运！」标签 + 银/蓝高亮 + 简短出现动画。
- **Lv3**：区域加「稀有！」标签 + 金/黄高亮 + 更强动画（如微光/缩放）。
- 幸运/稀有状态保持约 2s，再恢复或淡出，便于玩家注意到。

## 4. 实现要点
- `getItemLevel(itemId)`：判断物品在链上的等级 1/2/3。
- `state.lastGain = { itemId, qty, level }`，掷骰后写入并驱动上方区域。
- 棋盘上方 DOM：单一「本次获得」容器，根据 lastGain 与 level 切换样式类（普通 / gain-lucky / gain-rare）。
