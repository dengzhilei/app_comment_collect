# 快速开始指南

## 第一步：安装依赖

```bash
# 创建虚拟环境（推荐）
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# Linux/macOS:
source venv/bin/activate

# 安装依赖包
pip install -r requirements.txt

# 或者使用批处理文件（Windows推荐）
# 双击运行: 安装依赖_智能版.bat
```

## 第二步：配置游戏ID

编辑 `config.yaml` 文件，填入各游戏的实际ID：

### 如何查找App Store ID

1. 在浏览器中打开 App Store 或 iTunes
2. 搜索游戏名称
3. 打开游戏页面
4. 查看URL，格式类似：`https://apps.apple.com/app/id123456789`
5. 复制ID（如：123456789）

### 如何查找Google Play ID

1. 在浏览器中打开 Google Play Store
2. 搜索游戏名称
3. 打开游戏页面
4. 查看URL，格式类似：`https://play.google.com/store/apps/details?id=com.example.game`
5. 复制包名（如：com.example.game）

### 配置示例

```yaml
games:
  - name: "Carnival Tycoon"
    appstore_id: "123456789"           # 填入实际ID
    playstore_id: "com.example.game"   # 填入实际包名
    platforms: ["ios", "android"]
```

## 第三步：测试数据采集（可选）

在运行完整分析之前，可以先测试单个游戏的数据采集：

**方式1：使用批处理文件（推荐，Windows）**
- 双击运行 `运行测试.bat`

**方式2：命令行运行**
```bash
python3 test_scraper.py
```

测试脚本会：
- 验证应用ID是否正确
- 采集最近一年的评论（测试时限制为100条）
- 显示前3条评论示例
- 保存测试数据到 `data/raw/` 目录

## 第四步：运行完整分析

**方式1：使用批处理文件（推荐，Windows）**
- 双击运行 `运行完整分析.bat`

**方式2：命令行运行**
```bash
python3 main.py
```

程序将自动执行：
1. 数据采集（如果已有数据会询问是否重新采集）
2. 数据清洗
3. 情感分析
4. 内容分类
5. 生成报告

## 第五步：查看结果

- **分析报告**: `output/reports/` 目录
- **可视化图表**: `output/charts/` 目录
- **原始数据**: `data/raw/` 目录
- **处理后的数据**: `data/processed/` 目录

## 常见问题

### 问题1: 找不到游戏ID

**解决方案**:
- 确保游戏名称拼写正确
- 尝试在对应平台直接搜索
- 如果游戏已下架，可能无法采集

### 问题2: 采集失败或数据量少

**解决方案**:
- 检查网络连接
- 增加 `delay_between_requests` 的值（在config.yaml中）
- 检查游戏是否真的存在评论
- 某些游戏可能评论较少

### 问题3: 依赖安装失败

**解决方案**:
- 确保Python版本 >= 3.8
- 尝试使用国内镜像：`pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`
- 某些包可能需要编译工具（Windows用户可能需要安装Visual Studio Build Tools）

### 问题4: 中文显示乱码

**解决方案**:
- 确保系统支持UTF-8编码
- 在Windows上，可能需要设置环境变量：`set PYTHONIOENCODING=utf-8`
